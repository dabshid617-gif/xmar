# Page by Page Documentation

This document provides a detailed breakdown of each page in the application, including its functions, UI elements, and actions.

---

## 1. Index Page

*   **Page**: `Index`
*   **Route**: `/`
*   **Description**: This is the homepage of the application. It displays a search bar, a list of categories, and a list of featured products. It also includes a call to action for users to start selling.

### Functions

| Function Name             | Description                                                                                                                                 |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `loadProductsAndCategories` | Fetches the list of active products from the Supabase database. It also processes the product data to generate a list of categories with their counts. |
| `handleSearch`            | Triggered when the user submits the search form. It calls `loadProductsAndCategories` to filter the products based on the search query.      |
| `getCategoryIcon`         | Returns an icon component based on the category name.                                                                                       |

### UI Elements and Actions

| Element                   | Action/Redirect                                                                                                                             |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Search Input**          | Allows the user to enter a search query.                                                                                                    |
| **Search Button**         | Submits the search form and triggers the `handleSearch` function.                                                                           |
| **Location Button**       | A placeholder for location-based filtering. Currently, it does not have any functionality.                                                  |
| **Category Links**        | When a category is clicked, it navigates the user to the `/category/:categoryName` page, where they can see all the products in that category. |
| **"See all" Button**      | Navigates the user to the `/products` page, which is not yet implemented.                                                                   |
| **"Start Selling Now" Button** | Navigates the user to the `/products/new` page, which is the `ProductForm` for creating a new product.                                      |


---

## 2. Auth Page

*   **Page**: `Auth`
*   **Route**: `/auth`
*   **Description**: This page handles user authentication, including both sign-in and sign-up. It features a tabbed interface to switch between the login and registration forms.

### Functions

| Function Name | Description                                                                                                                                                           |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `handleAuth`  | Handles both the sign-in and sign-up logic. It calls the appropriate Supabase authentication function based on whether the user is logging in or signing up.          |

### UI Elements and Actions

| Element              | Action/Redirect                                                                                                                                                           |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Login/Sign Up Tabs** | Allows the user to switch between the login and sign-up forms.                                                                                                            |
| **Email Input**      | Input field for the user's email address.                                                                                                                                 |
| **Password Input**   | Input field for the user's password.                                                                                                                                      |
| **Full Name Input**  | (Sign-up form only) Input field for the user's full name.                                                                                                                   |
| **Username Input**   | (Sign-up form only) Input field for the user's desired username.                                                                                                            |
| **Sign In Button**   | Submits the login form and triggers the `handleAuth` function. On successful login, the user is redirected to the homepage (`/`).                                            |
| **Create Account Button** | Submits the sign-up form and triggers the `handleAuth` function. On successful sign-up, the user is redirected to the homepage (`/`).                                     |

---

## 3. Profile Page

*   **Page**: `Profile`
*   **Route**: `/profile/:userId`
*   **Description**: This page displays the profile of a user, including their username, full name, bio, and a list of the products they are selling. It also provides options to add a new product or message the user.

### Functions

| Function Name      | Description                                                                                                                                |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `checkCurrentUser` | Checks if there is a currently logged-in user.                                                                                             |
| `loadProfile`      | Fetches the profile information and the list of products for the user specified in the URL.                                                |
| `startChat`        | Initiates a chat with the user whose profile is being viewed. It navigates to the `/chat` page with the target user's ID in the state.        |

### UI Elements and Actions

| Element             | Action/Redirect                                                                                                                            |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **Add Product Button** | (Visible only on the user's own profile) Navigates to the `/products/new` page to create a new product.                                      |
| **Message Button**    | (Visible on other users' profiles) Initiates a chat with the user by calling the `startChat` function and navigating to the `/chat` page.     |
| **Edit Button**       | (Visible on the user's own products) Navigates to the `/products/:productId/edit` page to edit the selected product.                         |

---

## 4. Product Detail Page

*   **Page**: `ProductDetail`
*   **Route**: `/products/:productId`
*   **Description**: This page displays the detailed information of a single product, including its description, price, variants, and seller information. It also provides options to buy the product or contact the seller.

### Functions

| Function Name   | Description                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------- |
| `loadProduct`   | Fetches the detailed information of the product specified in the URL.                                   |
| `loadVariants`  | Fetches the variants available for the product.                                                         |

### UI Elements and Actions

| Element                 | Action/Redirect                                                                                                                                                                |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Variant Buttons**     | Allows the user to select a product variant. The price of the selected variant is displayed.                                                                                   |
| **Buy Now Button**      | (Placeholder) This button is intended to initiate the checkout process, but the functionality is not yet implemented.                                                          |
| **Order on WhatsApp Button** | Opens a new WhatsApp window with a pre-filled message to the seller, allowing the user to order the product directly. The seller's phone number is retrieved from their profile. |

## 5. Product Form Page

*   **Page**: `ProductForm`
*   **Route**: `/products/new` or `/products/:productId/edit`
*   **Description**: This page provides a form for creating a new product or editing an existing one. It includes fields for the product's title, description, price, image, status, and variants.

### Functions

| Function Name   | Description                                                                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `checkAuth`     | Checks if the user is authenticated and redirects to the `/auth` page if not.                                                                   |
| `loadProduct`   | Fetches the product data when the form is in edit mode.                                                                                         |
| `loadVariants`  | Fetches the variants of the product when the form is in edit mode.                                                                              |
| `uploadImage`   | Uploads an image file to Supabase Storage and returns the public URL.                                                                           |
| `handleSubmit`  | Handles the form submission. It creates or updates the product and its variants in the database and then navigates the user to their profile page. |

### UI Elements and Actions

| Element                 | Action/Redirect                                                                                                                                                                                             |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Product Title Input** | Input field for the product's title.                                                                                                                                                                        |
| **Description Textarea**| Text area for the product's description.                                                                                                                                                                    |
| **Price Input**         | Input field for the product's price.                                                                                                                                                                        |
| **Image URL Input**     | Input field for the product's image URL.                                                                                                                                                                    |
| **Image Input**         | Input field for uploading an image file.                                                                                                                                                                    |
| **Status Select**       | Dropdown for selecting the product's status (e.g., "active", "sold", "inactive").                                                                                                                            |
| **Add Variant Button**  | Adds a new set of input fields for a product variant.                                                                                                                                                       |
| **Remove Variant Button**| Removes a product variant.                                                                                                                                                                                  |
| **Create/Update Product Button** | Submits the form and triggers the `handleSubmit` function. On success, it navigates the user to their profile page (`/profile/:userId`).                                                              |
| **Cancel Button**       | Navigates the user back to the previous page.                                                                                                                                                               |
